# オブジェクト指向プログラミングの学習用サンプル

このディレクトリには、Python のオブジェクト指向プログラミングを学ぶためのサンプルプログラムが含まれています。

## 構成

### `anti_patttern/` ディレクトリ

各キャラクタータイプごとに別々のモジュールで実装した、冗長なコードの例です。

- `warrior.py`: 戦士キャラクターの実装
- `mage.py`: 魔法使いキャラクターの実装
- `archer.py`: 弓使いキャラクターの実装
- `main.py`: メインプログラム

**問題点**:

- 同じような処理を各モジュールで繰り返し書いている
- キャラクタータイプごとに異なる関数名を使う必要がある
- 新しいキャラクタータイプを追加する際に、多くのコードを書く必要がある

### `design_pattern/` ディレクトリ

クラスを使った設計で、共通の機能を再利用した実装です。

- `character.py`: キャラクターの基底クラス
- `warrior.py`: 戦士クラス（Character を継承）
- `mage.py`: 魔法使いクラス（Character を継承）
- `archer.py`: 弓使いクラス（Character を継承）
- `main.py`: メインプログラム

**利点**:

- 共通の機能は基底クラスに定義されているため、コードの重複がない
- すべてのキャラクターが同じメソッド名を使える（`attack()`, `get_status()`など）
- 新しいキャラクタータイプを追加する際も、基底クラスを継承するだけで簡単に追加できる

## 実行方法

### アンチパターン版を実行

```bash
cd anti_patttern
python main.py
```

### クラス版を実行

```bash
cd design_pattern
python main.py
```

## 学習のポイント

1. **継承**: 基底クラス`Character`を継承することで、共通の機能を再利用できます
2. **オーバーライド**: 各サブクラスで`get_status()`や`attack()`などのメソッドをオーバーライドして、キャラクターごとの特性を表現できます
3. **ポリモーフィズム**: 異なるクラスのインスタンスでも、同じメソッド名で呼び出せるため、コードがシンプルになります
4. **コードの再利用性**: クラスを使うことで、コードの重複を避け、保守しやすいコードになります
